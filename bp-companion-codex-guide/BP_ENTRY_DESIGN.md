# 极简血压录入设计

## 1. 用户目标

用户完成测量后，不需要先选择日期、输入来源或复杂表单。页面默认使用当前时间，并提供三个同等可见的入口：

```text
[手动输入]   [说出读数]   [拍血压计]
```

所有入口最终进入同一个确认草稿。识别结果不会自动保存。

## 2. 推荐页面

```text
记录血压                         刚刚 23:20  >

高压 / 收缩压                  [ 132 ]
低压 / 舒张压                  [  84 ]
心率（可选）                   [  70 ]

[ 🎙 说出读数 ]     [ 📷 拍血压计 ]

[ + 再记录一次 ]

                    [ 确认保存 ]
```

时间栏可点击进入日期和时间选择器。若不是今天，显示完整日期。

## 3. 状态机

```text
idle
 ├─ manual editing ───────────────┐
 ├─ recording → uploading → parse ├→ draft_ready → confirming → saving → saved
 └─ choosing photo → upload → OCR ┘                    ↘ error → editable draft
```

任何错误都保留手动编辑能力。离开页面前若有未保存内容，提示用户确认。

## 4. 时间规则

- 页面打开先显示当前时间。
- 用户第一次编辑、开始录音或选定照片时，再捕获一次当前时间作为默认测量时间。
- 用户可以手动改日期和时间。
- 提交 `measuredAt`、`timezone` 和 `clientRequestId`。
- 服务端统一转 UTC 存储，并回传本地显示值。
- 未来时间或很久以前的时间要求二次确认，不自动修改。

## 5. 语音规则

用户点击开始，再点击停止；最长 15 秒。页面示例：

```text
请说：高压一百三十二，低压八十四，心率七十
```

解析器支持：

- `高压 / 收缩压 / 上压`；
- `低压 / 舒张压 / 下压`；
- `心率 / 脉搏`；
- 中文数字、阿拉伯数字和常见停顿词。

解析器只做确定性抽取，不做诊断。缺少标签或存在冲突时，不猜测字段。

## 6. 拍照规则

使用单张压缩图片。拍摄提示：

- 屏幕占据画面主体；
- 避免反光；
- 数字保持正向；
- 尽量同时拍到 `SYS/DIA/PUL` 标签。

OCR 后依据标签、位置和单位匹配字段。背景中的时间、日期、设备编号等额外数字不能直接当作血压值。低置信度字段保持空白。

## 7. 隐私与留存

- 音频和图片只用于本次识别。
- 临时上传对象使用随机路径并设置短生命周期。
- 成功、失败、超时都执行删除。
- 原始媒体、文件 ID、签名 URL、转写文本和 OCR 全文不进入业务数据库或日志。
- 最终确认的读数按实施指南加密保存。
- 外部 ASR/OCR 使用情况必须写入小程序隐私说明。

## 8. 验收标准

- 手动录入三步以内完成。
- 语音或拍照完成后，用户只需核对并点击一次保存。
- 识别错误能够直接改字段，无需重新识别。
- 所有识别路径均不存在自动保存。
- 双击保存不会产生重复记录。
- 权限拒绝、取消、断网、供应商失败后仍能继续手动录入。
- 当前时间、手动时间和跨日/时区转换均通过自动化与真机测试。
